JOI libraries installed in namespace {{ include "joi.namespace" . }}:

{{ range $name, $lib := .Values.libraries }}{{ if $lib.enabled }}  - {{ $name }}  (oci://{{ $lib.image }}:{{ $lib.tag }})
{{ end }}{{ end }}
Each renders a Flux OCIRepository + a JaaS JsonnetLibrary of the same name. A
JsonnetSnippet references it in spec.libraries and imports it with the exact
jb vendor path used during local development:

  apiVersion: jaas.metio.wtf/v1
  kind: JsonnetSnippet
  metadata:
    name: example
    namespace: {{ include "joi.namespace" . }}
  spec:
    serviceAccountName: my-renderer
    files:
      main.jsonnet: |
        local g = import 'github.com/grafana/grafonnet/gen/grafonnet-latest/main.libsonnet';
        g.dashboard.new('demo')
    libraries:
      - kind: JsonnetLibrary
        name: grafonnet

my-renderer is a JaaS tenant ServiceAccount: it must exist in that namespace and
hold the JaaS tenant Role (read JsonnetLibrary, publish ExternalArtifact), or the
snippet's first reconcile fails with Forbidden. The Role is documented at:

  https://jaas.projects.metio.wtf/usage/tenancy-and-rbac/

Requires Flux source-controller (OCIRepository CRD) and JaaS in operator mode
(JsonnetLibrary CRD + reconciler) in the cluster.
