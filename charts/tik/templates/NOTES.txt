tik ({{ .Release.Name }}) installed in namespace {{ .Release.Namespace }}.

The read-only board / EDN API is served by:

  kubectl --namespace {{ .Release.Namespace }} port-forward svc/tik {{ .Values.ports.http }}:{{ .Values.ports.http }}
  # then open http://localhost:{{ .Values.ports.http }}/  (board)
  #      or   http://localhost:{{ .Values.ports.http }}/tickets.edn  (data)

The store lives on {{ if .Values.persistence.enabled }}the PersistentVolumeClaim {{ .Release.Name }}-store{{ else }}an emptyDir (EPHEMERAL — data is lost when the pod restarts; set persistence.enabled=true){{ end }}, mounted at /var/lib/tik.

{{ if .Values.ingest.enabled -}}
=== Mail ingestion ({{ .Values.ingest.protocol }}) ===

{{ if eq .Values.ingest.protocol "imap" -}}
The tik-ingest Deployment holds an IMAP connection open (IMAP IDLE) and ingests
mail as signed events the instant it arrives. It is loop-safe and idempotent.
{{- else -}}
The tik-ingest CronJob polls the POP3 mailbox on schedule "{{ .Values.ingest.schedule }}".
{{- end }}

It signs events with the key from Secret/{{ .Values.ingest.signingKeySecret.name }} and reads its
config from ConfigMap/{{ .Release.Name }}-ingest. The mailbox password is resolved from
{{ if .Values.ingest.passwordSecret.name }}Secret/{{ .Values.ingest.passwordSecret.name }} (mounted at /etc/tik/secrets){{ else }}the config's own secret spec{{ end }}.

With a ReadWriteOnce store the board and ingester must share one node — the
ingester prefers the board's node automatically. To scale the board across
nodes, set persistence.accessModes to [ReadWriteMany].
{{- else -}}
Mail ingestion is off (ingest.enabled=false). The board only serves derivations
of whatever events already exist in the store.
{{- end }}
